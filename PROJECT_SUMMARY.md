# MedHelp — Project Summary (snapshot: 2026-05-04)

## Overview

MedHelp is a full-stack medical triage app that uses a locally-hosted LLM (via Ollama) to interactively triage user symptoms through short yes/no questions and provide a likely diagnosis.

Components
- Backend: Node.js + Express (listens on port 3000). Key routes: `/login`, `/register`, `/profile`, `/ai/chat`, `/ai/reset`.
- Frontend: Angular (dev server at port 4200). Standalone components and routes include: Home, Login, Register, Dashboard, Chat.
- Database: MongoDB (user profiles stored in `Users` collection). Backend connects via MongoClient.
- AI: Ollama local model (configured via `OLLAMA_URL` and `OLLAMA_MODEL` in root `.env`). Model used: `cniongolo/biomistral` (BioMistral 7B) by default.

## Current Status

- Registration & login: working; JWT issued on login.
- User profile: stored in MongoDB and loaded into AI prompt for personalized triage.
- AI integration: `/ai/chat` calls the medical agent that builds a system prompt with user context, communicates with Ollama, and returns a reply.
- Chat UI: `ChatComponent` connects to `TriageService` which posts `{ email, message }` to `/ai/chat`. Responses use shape `{ reply, isFinal }`.
- Dev helper: `run-app.sh` launches backend and frontend together (uses ports 3000 & 4200).

## How to run locally

1. Ensure Ollama is running and the specified model is available on `http://localhost:11434` (or update `OLLAMA_URL` in `.env`).
2. Start MongoDB (or ensure Atlas connection string is set in `.env`).
3. From repo root:

```bash
# start both services (script already in repo)
./run-app.sh

# or start backend and frontend separately:
cd backend && npm install && npm start
cd frontend && npm install && npm start
```

4. Open UI: http://localhost:4200 — register, login, Dashboard → Start Triage → Chat.
